# RULES: Copilot 開発ルール

## 重要（守る）
1. 作りかけページを公開しない
   - 空ページ、準備中、近日公開、リンク切れ、404誘導はNG
2. arinkolab.com を全面リダイレクトしない
   - トップは必ず本文を返す
3. すべてのページに最低限の本文を置く
   - 「リンク集だけ」判定を避ける
4. ヘッダー/フッターのリンクは常に有効にする
   - Privacy/Terms/About/Contact は 404 にならない
5. 未確認の断定禁止：完了しました / デプロイ待機 / 本番反映済み 等を、実際にコマンド実行・確認していないのに書かない
6. 最終出力の必須構成：最後は必ず「ローカル検証手順（build/preview）」と「Cloudflare Pages の手動デプロイ手順」で締める

## ルーティング/URL
- 末尾スラッシュ運用（例: `/tools/`）
- 内部リンクは相対 or ルート相対で統一

## スタイル
- 初期は CSS 最小（後からTailwind等に差し替え可能な構造）
- 余白・見出し階層（H1→H2→H3）を丁寧に

## コンテンツ
- 日本語を基本
- 誇張しない（機能/保存有無/対応範囲は事実ベース）
- SQL Formatter ページには必ず:
  - できること
  - サンプル（整形前/後）
  - 注意（機密情報を入れない、免責、正しさ保証しない）

## ブログ導線
- ポータル側 `/blog/` は「案内ページ」にする（本文＋リンク）
- ブログ実体は `blog.arinkolab.com`（Hugo）

## 将来拡張の注意（覚書）
- 画像をJSで扱う機能（サムネ自動生成、モザイク/ぼかし、ウォーターマーク合成等）を追加する場合、
  media.arinkolab.com（R2）の CORS/直リンク制限/キャッシュ設計が論点になる可能性が高い。
