# RULES: Copilot 開発ルール

## 重要（守る）
1. **作りかけページを公開しない**
   - 空ページ、準備中、近日公開、リンク切れ、404誘導はNG
2. **arinkolab.com を全面リダイレクトしない**
   - トップは必ず本文を返す（AdSense 審査上の前提）
3. **すべてのページに最低限の本文を置く**
   - 「リンク集だけ」判定を避ける（段落 + 箇条書き + 注意事項など）
4. **ヘッダー/フッターのリンクは常に有効**
   - Privacy/Terms/About/Contact は 404 にならない

## UI/デザイン（今回の追加ルール）
- 色は `gemini提示案.html` のトークンを踏襲（CSS変数で定義）
- トップは三本柱（Business / General / Blog）をカードで見せる
- カードは初回表示で **左→右に少しスライド + フェードイン**
  - 3枚に軽いスタガー（遅延）
  - `prefers-reduced-motion: reduce` はアニメ無効
- **UIライブラリ追加は禁止**（Tailwind/React等は入れない）

## ルーティング/URL
- 末尾スラッシュ運用（例: `/tools/`）
- 内部リンクはルート相対で統一（例: `/tools/business/`）

## 文章/コンテンツ
- 日本語を基本
- 誇張しない（機能/保存有無/対応範囲は事実ベース）
- SQL Formatter ページには必ず:
  - できること
  - サンプル（整形前/後）
  - 注意（機密情報を入れない、免責、正しさ保証しない）

## ブログ導線
- ポータル側 `/blog/` は「案内ページ」（本文＋外部リンク）
- ブログ実体は `blog.arinkolab.com`（Hugo）

## 将来拡張の注意（覚書）
- 画像をJSで扱う機能（サムネ自動生成、モザイク/ぼかし、ウォーターマーク合成等）を追加する場合、
  media.arinkolab.com（R2）の CORS/直リンク制限/キャッシュ設計が論点になる可能性が高い。
